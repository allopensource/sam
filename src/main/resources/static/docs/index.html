What is Saga design pattern?
The Saga Pattern

One of the most well-known patterns for distributed transactions is called Saga. The first paper about it was published back in 1987, and it has been a popular solution since then. A Saga is a sequence of local transactions where each transaction updates data within a single service

Wikepedia

Our problem statement :

To develop SAGA pattern into a Transaction as a Service , where any application can register and process its transactions with a guarantee of completion towards commit or rollback.
